① データ構造

コンピュータは大量のデータを処理しますが、そのときデータの並べ方によって処理の効率が大きく変わります。この「データの並べ方」のことをデータ構造と呼びます。

データ構造は5種類
基本情報技術者試験では、以下の5つのデータ構造が出題されます。

(1) スタック ── 後入れ先出し
(2) キュー ── 先入れ先出し
(3) 配列 ── 表形式に並べる
(4) 連結リスト ── ポインタで数珠つなぎ
(5) 木構造 ── 階層構造を持つ

（1）スタック

スタックとは、最後に入れたデータを最初に取り出すデータ構造です。「積み上げられた本」をイメージしてください。本を積むときは上に重ね、取るときは一番上から取ります。途中の本を抜くことはできません。

スタックの操作用語

push：データをスタックに格納する（例：push C → Cを積む）
pop：データをスタックから取り出す（一番上のデータが出る）

この方式を後入れ先出し（LIFO：Last In First Out）といいます。
身近な例：
・Webブラウザの「戻る」ボタン → 最後に見たページに戻る（スタック構造）
・Ctrl+Zの「元に戻す」機能 → 最後の操作から順に取り消す

スタック体験デモ

pushで積み上げ、popで取り出してみよう

（2）キュー

キューとは、最初に入れたデータを最初に取り出すデータ構造です。「レジ待ちの列」をイメージしてください。先に並んだ人から会計を済ませます。

キューの操作用語

enqueue（エンキュー）：データをキューに入れる（列の最後に並ぶ）
dequeue（デキュー）：データをキューから取り出す（列の先頭から出る）

この方式を先入れ先出し（FIFO：First In First Out）といいます。
身近な例：
・プリンタの印刷待ち → 先に送った文書から印刷される
・コンビニのレジの列 → 先に並んだ人から会計する

キュー体験デモ

enqueueで並ばせ、dequeueで取り出してみよう
スタックとキューの違い ── 最重要ポイント

スタック：後入れ先出し（LIFO）→ 最後に入れたものから出る
キュー：先入れ先出し（FIFO）→ 最初に入れたものから出る

（3）配列

配列とは、データを表の形に並べたデータ構造です。個々のデータを要素、要素の位置を表す番号を要素番号（添字・インデックス）と呼びます。

1次元配列：要素が1列に並ぶ配列
→ 要素の取り出し：A[3]（配列Aの3番目の要素）

2次元配列：要素が縦横に並ぶ配列（表形式）
→ 要素の取り出し：A[2, 4]（2行目4列目の要素）
配列のメリット：
要素番号を指定するだけでデータに直接アクセスできる → 参照・更新が速い

（4）連結リスト

連結リストとは、データをポインタで数珠つなぎにしたデータ構造です。各要素は「データ本体」と「次の要素の位置（ポインタ）」のセットで構成されます。

ポインタとは？
次の要素のアドレス（主記憶装置上の位置）を指し示すものです。コンピュータはポインタをたどることで、目的のデータにたどりつきます。
連結リストのメリット：
データの挿入・削除の際、ポインタを書き換えるだけでOK → 挿入・削除が速い

配列の得意技

参照・更新が速い
要素番号で直接アクセスできるため

連結リストの得意技

挿入・削除が速い
ポインタを書き換えるだけのため
連結リストの種類

単方向リスト：先頭→末尾の一方向にだけデータをつなげる
双方向リスト：先頭⇔末尾の双方向にデータをつなげる
環状リスト：末尾と先頭がつながり、環（リング）状になっている

（5）木構造

木構造とは、階層構造（親子関係）を持つデータ構造です。植物の木を逆さにしたような形をしています。

木構造の用語

節（ノード）：枝分かれする箇所。データを保持する。
根（ルート）：最上位の節
葉（リーフ）：末端の節（子を持たない節）
親：分岐元の節 ／ 子：分岐先の節
2分木と2分探索木

2分木：親が最大2個の子を持つ木構造
2分探索木：2分木のうち、以下の条件を満たすもの：

左の子孫 ＜ 親 ＜ 右の子孫

上のSVG図がまさに2分探索木です。根「5」の左の子孫（1, 3, 4）はすべて5より小さく、右の子孫（7, 8, 9）はすべて5より大きくなっています。

メリット：データを探すとき、根から葉までの階層数だけ比較すれば見つかる → 効率的！
注意：「子」ではなく「子孫」
2分探索木の条件は「子」だけでなく、孫やひ孫など、その節より下のすべてのデータが条件を満たしている必要があります。これは試験でよく引っかけに使われるポイントです。

② アルゴリズムの基本

アルゴリズムとは、問題を解決するための手順のことです。同じ問題でも、効率の良い手順と悪い手順があります。コンピュータの世界では、最適なアルゴリズムを選ぶことがとても重要です。

アルゴリズムの4つの表現方法

アルゴリズムは次の4つの方法で表現できます：

(1) 文章 ── 文字で手順を書く
(2) 流れ図（フローチャート） ── 図で手順を書く ← 試験で最頻出！
(3) 数式 ── 関数で手順を書く
(4) プログラム言語 ── コンピュータが理解できる言語で書く

流れ図（フローチャート）

流れ図は、アルゴリズムを視覚的にわかりやすく表した図です。使用する記号はJIS（日本産業規格）で定められています。

流れ図で使う記号

端子（角丸長方形）：開始と終了を表す
処理（長方形）：計算や値の代入を表す。代入は「←」で表す
判断（ひし形）：条件分岐。出口に「Yes」「No」を書く
線：処理の順番を表す（必要に応じて矢印を付ける）
ループ端：繰り返し処理の開始と終了を表す
代入の表記：
「A ← B」は「Bの値をAに代入する」という意味。
例：「x ← x + 1」は「xの値に1を足して、その結果をxに入れ直す」です。

関数（function）

関数とは、複数の処理をひとまとめにしたものです。関数を定義しておけば、同じ処理を何度でも再利用できます。

関数の2つの重要用語

引数（ひきすう）：関数に渡す値（入力）
戻り値（もどりち）：関数が返す結果（出力）

例：絶対値を求める関数 f(-3) → 引数が「-3」、戻り値が「3」

プログラム言語での表現

基本情報技術者試験では、C言語やJavaなどの特定の言語の知識は不要です。以下の4つのキーワードの意味を把握しておけば読み解けます。

覚えるべき4つのキーワード

if：後ろに条件を書く（「もし〜ならば」）
then：後ろに条件が正しいときの処理を書く
else：後ろに条件が正しくないときの処理を書く
return：後ろに戻り値を書く（関数の結果を返す）
例：絶対値を求める関数

f(x): if x < 0 then return -x else return x

意味：「もしxが0より小さいなら、-xを返す。そうでなければxをそのまま返す」
試験での読み方のコツ：
1行にまとめて書かれることが多いですが、慌てずに if / then / else / return の区切りで分解して読みましょう。

トレース表

トレース表とは、流れ図の各処理を1つずつ実行したときの「変数の値」を書き出した表です。流れ図の問題を確実に解くための強力な武器です。

トレース表の使い方
① 処理に番号を振る
② 各処理を実行した後の変数の値を表に記入する
③ ループ処理は繰り返しごとに値を更新する

地味ですが、これを書けば流れ図の問題はほぼ確実に正答できます。

③ アルゴリズムの分類 ── 整列・探索・再帰

（1）整列アルゴリズム（ソートアルゴリズム）

データを特定の順番に並べるアルゴリズムです。試験ではクイックソートがほぼ毎回出題されます。

主な整列アルゴリズム

バブルソート：隣り合ったデータを比較・交換を繰り返す
選択ソート：最小値を選んで先頭と交換を繰り返す
挿入ソート：整列済み配列に未整列データを適切な場所に挿入
クイックソート：基準値で2グループに分割を繰り返す ← 最頻出！
クイックソートの手順

① 基準値を選ぶ（例：列の真ん中の数字）
② 基準値より小さい値のグループと大きい値のグループに分ける
③ 各グループの中でさらに基準値を選び、同じ手順を繰り返す
④ これ以上分割できなくなれば整列完了

ポイント：クイックソートは最も高速な整列アルゴリズムです。
例：[3, 5, 7, 6, 4, 2, 1, 9] を昇順に並べる
① 基準値 = 6 → 左グループ{3,5,4,2,1}、右グループ{7,9}
② 左グループの基準値 = 4 → {3,2,1} と {5}
③ さらに分割… → 最終的に [1,2,3,4,5,6,7,9] に整列！

（2）探索アルゴリズム

特定のデータを見つけ出すアルゴリズムです。ハッシュ法と2分探索法が頻出です。

主な探索アルゴリズム

線形探索法：先頭から順番に1つずつ照合する（単純だが遅い）
ハッシュ法：ハッシュ関数でデータの格納場所を特定する（最速）
2分探索法：範囲を半分ずつ絞り込んで探す（速い）
ハッシュ法のしくみ

① ハッシュ関数にデータを入力 → ハッシュ値が出力される
② ハッシュ値を要素番号として、ハッシュ表（配列）の該当箇所にデータを格納
③ 探索時も同じハッシュ関数を使えば、1回でデータの場所がわかる

代表的なハッシュ関数 = 剰余（mod）：割り算の余り
例：ハッシュ関数が「x mod 10」のとき、32のハッシュ値 = 32 mod 10 = 2

弱点：異なるデータが同じハッシュ値になる「衝突」が起こる可能性がある
2分探索法のしくみ

前提：データがあらかじめ昇順（または降順）に整列されている必要がある

① 中央の要素を取り出し、探索対象と比較
② 探索対象のほうが大きい → 右半分に絞る
③ 探索対象のほうが小さい → 左半分に絞る
④ この操作を繰り返して目的のデータを見つける

2つの特徴：
【特徴1】データをあらかじめ整列させておく必要がある
【特徴2】データを2つに分けながら探索する

（3）再帰的アルゴリズム

再帰的アルゴリズムとは、処理の途中で自分自身を呼び出すアルゴリズムです。「再帰」とは「繰り返し」という意味で、自分自身を呼び出すことを再帰呼び出しといいます。

再帰が使われる3つの典型パターン

総和：1からnまでの合計（Σ）
剰余：割り算の余り（mod）
階乗：1からnまでの積（n! = n × (n-1) × … × 1）
例：総和を求める再帰関数

f(x): if x ≦ 1 then return 1 else return x + f(x - 1)

f(5) = 5 + f(4)
     = 5 + 4 + f(3)
     = 5 + 4 + 3 + f(2)
     = 5 + 4 + 3 + 2 + f(1)
     = 5 + 4 + 3 + 2 + 1 = 15
例：階乗を求める再帰関数

F(n): if n = 0 then return 1 else return n × F(n - 1)

F(3) = 3 × F(2)
     = 3 × 2 × F(1)
     = 3 × 2 × 1 × F(0)
     = 3 × 2 × 1 × 1 = 6
試験での解き方のコツ：
再帰関数が出たら、実際に具体的な数値を代入して計算するのが最も確実です。いきなり一般化しようとせず、1ステップずつ展開していきましょう。

オーダー（計算量）

アルゴリズムの効率を比較するために、オーダー（計算量）という指標を使います。O記法（ビッグオー記法）で表します。

各アルゴリズムのオーダー（データ数 n）

ハッシュ法：O(1) ── 常に1回で見つかる（最速）
2分探索法：O(log n) ── 半分ずつ絞るので効率的
線形探索法：O(n) ── 最悪全部調べる
クイックソート：O(n log n) ── 分割×比較
log（対数）の計算方法

「log2 8」とは「2を何回かけたら8になるか？」という意味。
23 = 8 なので、log2 8 = 3

つまり、データが8個のときの2分探索法の計算量は最大3回の比較で済みます。

④ プログラムの性質と種類

プログラムの4つの性質

プログラムにはさまざまな性質があります。試験では再帰と再入可能の内容が問われます。

試験で内容が問われる2つ

再帰（リカーシブ）：プログラム実行中に自分自身を呼び出せる性質
→ 再帰プログラムは、自分の中で自分を使える

再入可能（リエントラント）：複数のプログラムから同時に呼び出されても正しく動作する性質
→ AとBが同時にCを呼び出しても、Cがそれぞれに正しい結果を返せる
残りの2つ（名前と意味の対応を覚えるだけでOK）

再配置可能（リロケータブル）：主記憶のどこに置いても実行できる性質
再使用可能（リユーザブル）：一度メモリに格納すれば、再読み込みなしで何度でも実行できる性質
暗記のコツ：「リ」で始まる4つ
リカーシブ（再帰）、リエントラント（再入可能）、リロケータブル（再配置可能）、リユーザブル（再使用可能）

名称に「Java」が含まれる用語

Javaは最も有名なプログラム言語の1つです。名前に「Java」がつく用語が複数あり、混同しやすいため試験によく出ます。

Java系用語の整理

Java：機種やOSに依存しないプログラム言語

Javaアプレット：サーバからダウンロードしてクライアント側のWebブラウザ上で実行されるJavaプログラム

Javaサーブレット：サーバ上で実行される、動的なWebページを作るJavaプログラム

JavaBeans：よく使われる機能を部品化・再利用できるようにする仕様

JavaScript：動的なWebページを作るための言語。Javaとはまったくの別物！

Javaアプレット

（ブラウザ上で実行）

⑤ その他の言語 ── HTML・XML・CSS・Ajax

第3章の最後のテーマです。計算問題がなく、用語と役割の対応を覚えるだけで得点できます。非常にコスパの良い分野です。

HTML

HTML（HyperText Markup Language）は、Webページを作成するマークアップ言語です。タグ（< >で囲まれた要素）を使って、Webページの論理構造（どのような要素があり、どういう関係か）を指定します。

HTMLの例：
<html>
  <title>ここはタイトル</title>
  <body>ここは本文</body>
</html>
ポイント：HTMLではタグの使い方があらかじめ決まっている（勝手にタグを作れない）

XML

XML（eXtensible Markup Language）は、自分でタグを定義できるマークアップ言語です。Web上でのデータ交換に使われます。

XMLの例：
<生徒>
  <名前>山田太郎</名前>
  <電話番号>090-0000-0000</電話番号>
</生徒>

HTML

目的：Webページ作成
タグ：使い方が決まっている

XML

目的：データ交換
タグ：自分で定義できる

CSS

CSS（Cascading Style Sheets）は、HTML文書のデザイン（視覚表現）を指定する言語です。文字の大きさ、色、行間、レイアウトなどを指定します。

HTMLとCSSの役割分担

HTML：Webページの内容と構造を指定（「何を表示するか」）
CSS：Webページの見た目を指定（「どう表示するか」）

CSSのキーワードは「視覚表現の情報」です。この言葉が選択肢に出たらCSS！

Ajax

Ajax（Asynchronous JavaScript and XML）は、Webページの画面を切り替えることなく、動的なユーザーインタフェースを実現する技術です。

同期通信 vs 非同期通信

同期通信：サーバの処理が完了するまで待つ。画面全体が切り替わる。
非同期通信：サーバの処理完了を待たずに次の処理に進む。画面は切り替わらない。

Ajaxの「A」はAsynchronous（非同期）の頭文字。
「非同期通信」「画面遷移を伴わない」と来たらAjax！
Ajaxの代表例：Googleマップ
地図をドラッグすると、ページ全体を再読み込みせずに新しい地図が表示されます。これがAjaxの技術です。
第3章の最重要まとめ

・スタック = LIFO（後入れ先出し）、キュー = FIFO（先入れ先出し）
・配列 = 参照・更新が速い、連結リスト = 挿入・削除が速い
・2分探索木の条件：左の子孫 < 親 < 右の子孫
・クイックソート = 基準値で2グループに分割
・ハッシュ法 O(1)、2分探索法 O(log n)、線形探索法 O(n)
・再帰＝自分自身を呼び出す、再入可能＝複数から同時に呼ばれてもOK
・HTML＝Webページ作成、XML＝データ交換（タグ自由）、CSS＝デザイン
・Ajax＝非同期通信で画面遷移なしの動的UI
