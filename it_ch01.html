① データの単位

ビットとバイト

コンピュータは全ての情報を0と1の2種類の数字で表現します。この0または1の1桁分をビット（bit）と呼びます。

ビット（bit）：情報の最小単位。0か1のどちらかを表す。
バイト（Byte）：8ビット = 1バイト。半角英数字1文字分。

nビットで表せる状態の数

nビットで表せる状態の数 = 2n

1ビット → 21 = 2通り（0, 1）
2ビット → 22 = 4通り（00, 01, 10, 11）
3ビット → 23 = 8通り（000〜111）
8ビット（1バイト）→ 28 = 256通り
試験で頻出：「32ビットで表現できるパターン数は24ビットの何倍か？」→ 232 ÷ 224 = 28 = 256倍

データの大きさの単位

K（キロ）= 約1,000（正確には1,024 = 210）
M（メガ）= 約100万（1,024K）
G（ギガ）= 約10億（1,024M）
T（テラ）= 約1兆（1,024G）

② 基数変換

10進数・2進数・16進数

10進数：値が10になると桁が上がる（0〜9を使う）。普段の生活で使う。
2進数：値が2になると桁が上がる（0と1だけ）。コンピュータ内部の表現。
16進数：値が16になると桁が上がる（0〜9とA〜Fを使う）。メモリアドレスなどに使用。
10進数の「10」は…
・2進数では「1010」
・16進数では「A」

桁の重み

桁の重みとは、各桁の位置が持つ値の大きさのことです。

10進数の桁の重み：
7297 = 7×103 + 2×102 + 9×101 + 7×100
= 7000 + 200 + 90 + 7

2進数の桁の重み：
1011 = 1×23 + 0×22 + 1×21 + 1×20
= 8 + 0 + 2 + 1 = 11（10進数）

2進数 → 10進数

各桁に桁の重み（2n）をかけて、全部たし合わせる。
2進数「10110」を10進数に変換：
1×24 + 0×23 + 1×22 + 1×21 + 0×20
= 16 + 0 + 4 + 2 + 0 = 22

10進数 → 2進数

整数部：2で割り続けて、余りを下から読む。
小数部：2をかけ続けて、整数部を上から読む。
10進数「22」を2進数に変換：
22÷2=11 余り0
11÷2=5 余り1
5÷2=2 余り1
2÷2=1 余り0
1÷2=0 余り1
↑下から読む → 10110

2進数 ↔ 16進数

2進数を4桁ずつ区切って、それぞれを16進数に変換する。
2進数「10110100」→ 「1011 | 0100」→ 「B4」（16進数）
16進数「3F」→ 「0011 | 1111」→ 「00111111」（2進数）

③ 負の2進数とシフト演算

負の2進数の表し方

2進数では「-」記号を使わず、別の方法でマイナスを表現します。

2の補数：負の2進数を表す最も一般的な方法。

作り方：
① すべてのビットを反転する（0→1, 1→0）
② 反転した結果に1を加える

例：8ビットで「-2」を表す
2 = 00000010
①反転 → 11111101
②+1 → 11111110（これが-2）
符号ビット：先頭のビットで正負を判別
・先頭が0 → 正の数
・先頭が1 → 負の数

シフト演算

ビット列を左右にずらすことで、かけ算・わり算を高速に行えます。

論理シフト：符号を考慮せず、単純にビットをずらす。空いた場所には0を入れる。

算術シフト：符号ビット（先頭）を固定したまま、残りのビットをずらす。負の数に使う。
左に1つシフト = ×2（2倍）
右に1つシフト = ÷2（1/2倍）

左に3つシフト = ×23 = ×8
右に5つシフト = ÷25 = ÷32
試験で頻出：「正の整数xを2ビット左にシフトして、xを加える」→ x×4 + x = 5x
